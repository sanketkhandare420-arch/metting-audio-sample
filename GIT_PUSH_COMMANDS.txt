# Quick GitHub Push Commands

Run these commands in PowerShell (in your project directory):

```powershell
cd "C:\Users\USER\Desktop\metting audio sample"

# Configure git (one-time setup)
git config user.name "sanketkhandare420-arch"
git config user.email "your-email@example.com"

# Initialize repository
git init

# Add all files
git add .

# Create first commit
git commit -m "Initial commit: Audio Action-Item Extractor with live recording, translation, and history"

# Add GitHub remote
git remote add origin https://github.com/sanketkhandare420-arch/metting-audio-sample.git

# Rename branch to main (if needed)
git branch -M main

# Push to GitHub
git push -u origin main
```

**Note:** You'll be prompted for GitHub credentials. Use:
- Username: `sanketkhandare420-arch`
- Password: (use Personal Access Token, see below)

## Generate GitHub Personal Access Token (PAT)

If you don't have a token:

1. Go to https://github.com/settings/tokens
2. Click "Generate new token" → "Generate new token (classic)"
3. Name: `Railway Deployment`
4. Check: `repo` (full control of repositories)
5. Click "Generate token"
6. **Copy and save the token** (you won't see it again)
7. Use this token as your password when pushing

## Alternative: Use SSH (No Password Needed)

```powershell
# Generate SSH key (if you don't have one)
ssh-keygen -t ed25519 -C "your-email@example.com"
# Press Enter 3 times to accept defaults

# Add SSH key to GitHub:
# 1. Go to https://github.com/settings/ssh/new
# 2. Run: Get-Content "$env:USERPROFILE\.ssh\id_ed25519.pub" | Set-Clipboard
# 3. Paste in GitHub
# 4. Name it "Windows Machine"

# Use SSH remote instead:
git remote remove origin
git remote add origin git@github.com:sanketkhandare420-arch/metting-audio-sample.git

# Push
git push -u origin main
```

Done! Your code is on GitHub and ready to deploy to Railway.
